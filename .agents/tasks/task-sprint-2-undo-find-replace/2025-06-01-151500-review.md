# Undo/Redo and Find & Replace for Phase 4 editor (v2)

Two capabilities added to the translation review pipeline: a cell-level undo stack that tracks translation and approved-status edits, and a global find-and-replace system exposed via both a modal dialog and a CLI command. The undo stack is a lightweight index-pointer implementation (`UndoStack` as QObject emitting a signal on mutation) wired into `_EntriesModel.setData()` with a guard flag to prevent recursion during replay. Find & replace is factored into a shared utility module consumed by both the GUI dialog and `stx replace`. 50 new tests cover the feature surface.

**Watch for:** The `_applying_undo` flag remains fragile if copied to new call sites ŌĆö only exception-safe within `_apply_command_value` (confirmed, low risk since usage is single-site).

**Verdict**: APPROVED

## High-level view

All five issues from v1 have been addressed. The GUI scope combo is now restricted to "Translations Only" (the `_SCOPE_LABELS` list is a single-element array), eliminating the silent-drop-of-non-translation-replacements problem. The undo stack enforces a 500-command cap with oldest-first eviction in `push()`. The page-switch signal (`self._stack.currentChanged`) is wired to `_refresh_undo_actions` in `main_window.py`, so Edit menu items correctly disable when navigating away from Phase 4. "Apply to all rows" now iterates `doc.entries` with exact equality comparison (`entry.translation == old_text`) instead of routing through `compute_replacements`'s substring regex.

The `_applying_undo` guard in `_apply_command_value` uses try/finally, preventing a stuck flag on exceptions during replay. The one remaining observation is that bulk Replace All pushes N individual undo commands rather than a single compound command, making full revert require N Ctrl+Z presses ŌĆö a UX trade-off rather than a correctness issue.

## Bulk replace as individual undo commands

Each replacement from Find & Replace pushes a separate `UndoCommand`. For a 200-match Replace All, that's 200 Ctrl+Z presses to fully revert. A compound-command (wrapping N commands into one undo step) would be the typical solution if users report friction. Given the 500-command cap and the typical 1-20 match counts for translation work, this is unlikely to cause practical problems.

## Test coverage for fixes

The new tests cover: scope combo restriction (`test_scope_combo_only_has_translations`), stack cap (`test_max_size_evicts_oldest`, `test_max_size_preserves_newest`), and exact-match behaviour (`TestApplyToAllRowsExactMatch`). No test drives a full `_goto()` cycle and asserts menu action state after page-switch ŌĆö the wiring is confirmed by code inspection but not exercised end-to-end.
